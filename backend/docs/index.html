Документация PhotoBank Backend API

REST API для фотобанка

Версия: 1.0.0

Обзор Авторизация Изображения Избранное Модели данных

Обзор

FastAPI приложение, предоставляющее REST API для фотобанка:

  • Регистрация и авторизация пользователей (JWT)
  • CRUD операции для изображений
  • Управление избранными фотографиями
  • Поиск и фильтрация изображений

Авторизация

POST /register

Регистрация нового пользователя

Создаёт нового пользователя в системе с хешированием пароля. Проверяет уникальность имени и минимальную длину пароля.

Параметры:

  • user_data:UserCreate - Данные для регистрации

Возвращает:

UserResponse - Данные созданного пользователя

Пример:

POST /register
{
    "username": "john",
    "password": "secret123"
}
POST /login

Авторизация пользователя

Проверяет учётные данные и выдаёт JWT токен.

Параметры:

  • user_data:UserLogin - Данные для входа

Возвращает:

Token - JWT токен доступа

Пример:

POST /login
{
    "username": "john",
    "password": "secret123"
}
# Response: {
    "access_token": "eyJ...",
    "token_type": "bearer",
    "username": "john"
}
GET /me

Получение данных текущего пользователя

Возвращает информацию об авторизованном пользователе. Требует авторизации.

Параметры:

  • current_user:str - Имя пользователя (из токена)

Возвращает:

UserResponse - Данные пользователя

Требует:

Authorization header с JWT токеном

Пример:

GET /me
Authorization: Bearer eyJ...

Изображения

GET /images

Получение списка изображений

Возвращает все изображения или фильтрует по поисковому запросу. Для авторизованных пользователей показывает статус "в избранном".

Параметры:

  • search:Optional[str] - Поисковый запрос (опционально)
  • current_user:Optional[str] - Имя текущего пользователя (опционально)

Возвращает:

List[Image] - Список изображений

Пример:

GET /images
GET /images?search=горы
POST /upload

Загрузка нового изображения

Создаёт новую фотографию в базе данных. Требует обязательной авторизации.

Параметры:

  • image_data:ImageCreate - Данные изображения
  • current_user:str - Имя пользователя (из токена)

Возвращает:

Image - Созданное изображение

Требует:

Authorization header с JWT токеном

Пример:

POST /upload
Authorization: Bearer eyJ...
{
    "url": "https://...",
    "title": "Фото",
    "tags": ["tag1"]
}
DELETE /images/{image_id}

Удаление изображения

Удаляет фотографию из базы данных. Только владелец может удалить своё изображение.

Параметры:

  • image_id:str - ID изображения
  • current_user:str - Имя пользователя (из токена)

Возвращает:

dict - Сообщение об успешном удалении

Требует:

Authorization header с JWT токеном

Пример:

DELETE /images/123
Authorization: Bearer eyJ...
PUT /images/{image_id}

Обновление изображения

Обновляет название, описание или теги изображения. Только владелец может редактировать своё изображение.

Параметры:

  • image_id:str - ID изображения
  • image_data:ImageUpdate - Данные для обновления
  • current_user:str - Имя пользователя (из токена)

Возвращает:

Image - Обновлённое изображение

Требует:

Authorization header с JWT токеном

Пример:

PUT /images/123
Authorization: Bearer eyJ...
{
    "title": "Новое название",
    "tags": ["new"]
}
GET /user/{username}/images

Получение изображений конкретного пользователя

Возвращает все фотографии указанного пользователя.

Параметры:

  • username:str - Имя пользователя
  • current_user:Optional[str] - Текущий пользователь (опционально)

Возвращает:

List[Image] - Список изображений пользователя

Пример:

GET /user/admin/images

Избранное

POST /like

Добавление/удаление изображения в избранное

Управляет списком избранных фотографий пользователя.

Параметры:

  • action:LikeAction - Действие (ID фото и статус)
  • current_user:str - Имя пользователя (из токена)

Возвращает:

dict - Сообщение об успешном выполнении

Требует:

Authorization header с JWT токеном

Пример:

POST /like
Authorization: Bearer eyJ...
{
    "image_id": "123",
    "is_liked": true
}
GET /favorites

Получение списка избранных изображений

Возвращает все фотографии, добавленные в избранное пользователем.

Параметры:

  • current_user:str - Имя пользователя (из токена)

Возвращает:

List[Image] - Список избранных изображений

Требует:

Authorization header с JWT токеном

Пример:

GET /favorites
Authorization: Bearer eyJ...
GET /user/{username}/favorites

Получение избранных фотографий пользователя

Пользователь может просматривать только свои избранные фото.

Параметры:

  • username:str - Имя пользователя
  • current_user:str - Имя текущего пользователя (из токена)

Возвращает:

List[Image] - Список избранных изображений

Требует:

Authorization header с JWT токеном

Пример:

GET /user/john/favorites
Authorization: Bearer eyJ...

Модели данных

UserCreate

Модель для регистрации нового пользователя

  • username: str - Имя пользователя (мин. 3 символа)
  • password: str - Пароль пользователя (мин. 6 символов)

UserLogin

Модель для входа пользователя

  • username: str - Имя пользователя
  • password: str - Пароль пользователя

UserResponse

Модель ответа с данными пользователя

  • username: str - Имя пользователя

Token

Модель JWT токена

  • access_token: str - JWT токен доступа
  • token_type: str - Тип токена (bearer)
  • username: str - Имя пользователя

Image

Модель изображения

  • id: str - Уникальный идентификатор
  • url: str - URL изображения
  • title: str - Название фотографии
  • description: str - Описание фотографии
  • author: str - Имя автора
  • author_id: str - ID автора
  • tags: List[str] - Список тегов
  • publishDate: str - Дата публикации (ISO 8601)
  • isLiked: bool - Статус "в избранном"

ImageCreate

Модель для создания нового изображения

  • url: str - URL изображения
  • title: str - Название фотографии
  • description: str - Описание фотографии
  • tags: List[str] - Список тегов

ImageUpdate

Модель для обновления изображения

  • title: Optional[str] - Новое название (опционально)
  • description: Optional[str] - Новое описание (опционально)
  • tags: Optional[List[str]] - Новые теги (опционально)

LikeAction

Модель действия "Избранное"

  • image_id: str - ID изображения
  • is_liked: bool - Статус (True - добавить, False - удалить)
